# Cofit AI Agent Demo

Extracted from `ai-skill-platform`'s `/ai-brain` endpoint. Supports A2A (Agent-to-Agent) multi-expert orchestration.

## Architecture

### v1 (Single Agent)
```
BE ──POST /ai-brain──> FastAPI ──> ADK Agent (Gemini) ──> return result
```

### v2 (Orchestrator + Sub-agents)
```
BE ──POST /ai-brain──> FastAPI ──> Orchestrator (gemini-pro)
                                        │
                                        ├── Sub-agent: lab_report (gemini-flash-lite)
                                        ├── Sub-agent: body_measurement (gemini-flash-lite)
                                        └── Sub-agent: orders (gemini-flash-lite)
                                        │
                                        └── aggregate results ──> return
```

**Three orchestration modes:**
- `auto`: ADK automatically routes to the appropriate sub-agent (default)
- `parallel`: all run concurrently, orchestrator aggregates
- `sequential`: run in order, passing previous result to next step

## Quick Start

```bash
# Install dependencies
pip install -r requirements.txt

# Set environment variables
export GCP_PROJECT_ID=cofit-stg
export COFIT_API_URL=https://staging.cofit.me  # required for v2 remote skill
export COFIT_TOKEN=your-token                   # required for v2 remote skill

# Run locally
python main.py
# or
uvicorn main:app --reload --port 8080
```

## API

### POST /ai-brain

**No auth required** — internal BE call.

#### v1 Request (Single Agent):
```json
{
  "key": "doctor_visit_initial",
  "system_prompt": "You are an experienced physician...",
  "model_config": { "model": "gemini-flash" },
  "context_data": {
    "client": { "real_name": "Samuel", "gender": "male", "age": 35 }
  },
  "message": "Please generate an initial consultation report for this patient",
  "stream": false
}
```

#### v2 Request (Orchestrator + Sub-agents):
```json
{
  "key": "clinic_brain_v2",
  "system_prompt": "You are the clinic AI brain. Delegate to the appropriate expert based on the question...",
  "model_config": { "model": "gemini-pro" },
  "context_data": { "client_id": 351 },
  "message": "How are this client's recent lab results and weight changes?",
  "skills": [
    { "skill_key": "lab_report", "description": "Analyze lab report data, identify abnormal values and provide recommendations" },
    { "skill_key": "body_measurement", "description": "Analyze body composition and physical data trends" },
    { "skill_key": "orders", "description": "Query and analyze client order history" }
  ],
  "orchestration_mode": "auto",
  "stream": false
}
```

Each skill supports two modes:
- **Minimal mode**: pass only `skill_key` + `description`, AI platform fetches config from BE automatically
- **Inline mode**: pass `system_prompt`, `model_config`, `context_data` directly, no BE call needed

#### Response (non-streaming):
```json
{
  "result": "Hi Samuel! Based on your test report...",
  "skill_key": "clinic_brain_v2"
}
```

#### Response (streaming, `stream: true`):
```
data: {"text": "Hi Samuel!"}
data: {"text": "Based on your..."}
data: [DONE]
```

### POST /v1/agents/{agent_key}/run

Run a multi-skill agent by key. Fetches manifest and context data from BE automatically.

```json
{
  "client_id": 351,
  "message": "Please generate a comprehensive analysis report",
  "stream": false
}
```

### GET /health

Health check.

### GET /docs

Swagger API docs (auto-generated by FastAPI).

## Backward Compatibility

| Scenario | Behavior |
|----------|----------|
| No `skills` passed | Same as v1 single agent |
| `skills` + `auto` | ADK sub_agent auto routing |
| `skills` + `parallel` | All run concurrently, orchestrator aggregates |
| `skills` + `sequential` | Run in order, context chained between steps |
